# Label Catalog

Document the repository's lifecycle and supporting labels in one place.

## Lifecycle Labels

### `ready-for-build`

- meaning: issue is ready for readiness validation

### `in-progress`

- meaning: implementation is authorized

### `in-review`

- meaning: implementation is complete enough for review

### `done`

- meaning: merged and complete

## Supporting Labels

Add the repository's local labels here.

For each label, document:

- meaning
- whether it is routing, ownership, risk, or topology related
- whether automation depends on it

## Notes

- lifecycle labels represent state
- supporting labels should not replace lifecycle state
- topology labels, if used, should stay separate from lifecycle labels
